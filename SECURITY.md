# Security Policy

## Supported Versions

The following versions currently receive security updates:

| Version | Supported |
| ------- | --------- |
| v1.x    | ✅ Yes     |
| v0.x    | ❌ No      |

---

## Reporting a Vulnerability

If you discover a security vulnerability in AutoRec, please report it responsibly.

### Please Include

* Description of the vulnerability
* Affected version(s)
* Steps to reproduce
* Impact assessment
* Proof of concept (if applicable)

Providing detailed information helps us investigate and resolve issues more efficiently.

---

## Responsible Disclosure

Please do **not** publicly disclose security vulnerabilities until they have been reviewed and addressed.

We ask researchers to:

* Avoid public disclosure before remediation
* Give maintainers reasonable time to investigate
* Provide sufficient technical details for reproduction
* Avoid accessing or modifying data beyond what is necessary to demonstrate the issue

---

## Scope

This policy applies to:

* AutoRec source code
* Dashboard components
* Reporting components
* Docker deployment files
* Database layer
* Reconnaissance modules

This policy does not apply to:

* Third-party tools used by AutoRec
* External services
* User-created modules
* Misconfigurations in systems scanned by AutoRec

---

## Security Considerations

AutoRec integrates with several third-party reconnaissance tools.

Users should:

* Keep dependencies updated
* Verify tool integrity before installation
* Use AutoRec only on systems they own or are authorized to assess
* Follow applicable laws and regulations

---

## Supported Reporting Channels

At this time, security issues may be reported through:

* GitHub Security Advisories (preferred)
* Private communication with project maintainers

Public GitHub issues should not be used for reporting undisclosed security vulnerabilities.

---

## Response Process

Reported vulnerabilities will typically follow this process:

1. Acknowledgement
2. Investigation
3. Validation
4. Remediation
5. Release of a fix
6. Public disclosure (when appropriate)

---

## Safe Harbor

We support responsible security research conducted in good faith.

Researchers acting in accordance with this policy and applicable laws will be treated as collaborators in improving the security of the project.

---

Thank you for helping keep AutoRec secure.
